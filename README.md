## 🎯 미션 개요
크리스마스 이벤트 프로그램

<br>

## 📝 기능 요구 사항

### 1. 할인
[ ] 일반 할인 : 12/1 ~ 12/25
- 1000원으로 시작해서 하루마다 100원씩 증가한 금액 할인
[ ] 평일 할인 : 일, 월, 화, 수, 목
- 디저트 메뉴 개당 2,023원 할인
[ ] 주말 할인 : 금, 토
- 메인 메뉴 개당 2,023원 할인
[ ] 특별 할인 : 달력에 별이 있는 경우 (일요일 or 크리스마스)
- 1000원 할인
[ ] 일반 할인을 제외한 다른 할인은 12월 내내 적용

### 2. 증정 이벤트
[ ] 샴페인 증정 (1개) : 할인 전 총주문 금액이 12만원 이상일 때

### 3. 이벤트 배지 부여
[ ] 총혜택 금액에 따라 부여
- 5000원 이상 : 별
- 10000원 이상 : 트리
- 20000원 이상 : 산타

### 4. 고객에게 안내할 이벤트 주의 사항
[ ] 총주문 금액 10000원 이상부터 이벤트 적용
[ ] 음료만 주문 시, 주문 불가
[ ] 메뉴는 한 번에 최대 20개까지만 주문 가능

### 5. 개발 요청 사항
[ ] 고객은 날짜 및 메뉴를 미리 선택
[ ] 이벤트 플래너는 아래의 내용을 보여줌
- 주문 메뉴
- 할인 전 총 주문 금액
- 증정 메뉴
- 해당하지 않을 시 "없음" 출력
- 혜택 내역 (적용된 이벤트 내역)
- 해당하지 않을 시 "없음" 출력
- 총혜택 금액 (할인 금액의 합계 + 증정 메뉴의 가격)
- 할인 후 예상 결제 금액 (할인 전 총주문 금액 - 할인 금액)
- 12월 이벤트 배지 내용
- 해당하지 않을 시 "없음" 출력


### 6. 기본 예외 처리
[ ] 잘못된 입력 시 `IllegalArgumentException`을 발생시킨다.
[ ] 모든 에러 메세지는 `[ERROR]`로 시작하는 에러 메시지를 출력한다. 이때 `Exception`이 아닌 `IllegalArgumentException`, `IllegalStateException` 등과 같은 명확한 유형을 처리한다.


### 7. 세부 예외 처리
[ ] 방문 날짜는 1 이상 31 이하의 숫자
-  `[ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요.` (위 1개의 예외 처리 메세지)
[ ] 메뉴판에 없는 메뉴는 입력할 수 없다
[ ] 메뉴의 개수는 1 이상의 숫자만 입력 가능
[ ] 메뉴 형식은 예시와 동일해야한다.
[ ] 중복 메뉴 입력 불가, 같은 메뉴 여러개 주문 시 수량을 늘려야함.
- `[ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.` (위 4개의 예외 처리 메세지)

## 🛠️ 프로그래밍 요구 사항

### 실행 환경
- JDK 버전: 21
- 프로그램 시작점: `Application.main()`
- 빌드 도구: Gradle

### 제약 사항
- `build.gradle` 파일 변경 금지
- 외부 라이브러리 추가 금지
- `System.exit()` 사용 금지
- 입력과 출력을 담당하는 클래스를 별도로 구현
- 사용자가 입력하는 값은 `camp.nextstep.edu.missionutils.Console`의 `readLine()`을 활용


### 코드 컨벤션
- 기본적으로 [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html) 준수
- 추가로 [우아한테크코스 Java 코딩 컨벤션](https://github.com/woowacourse/woowacourse-docs/tree/main/styleguide/java) 준수
- `indent depth` ≤ 2
- `else` 및 `switch/case` 사용 금지
- 삼항 연산자 금지
- 메서드 길이는 15라인 이하
- 메서드는 한 가지 일만 수행
- 단위 테스트 작성
